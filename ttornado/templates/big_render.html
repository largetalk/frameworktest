2. 生产环境部署¶

2.1 生产服务器角色分配¶

目前线上采用3台虚拟机作为生产服务器, F5做负载均衡. 三台服务器均安装 Ubuntu 11.10 Server 64bit 操作系统.

服务器配置为 8核Cpu 8G内存 30G硬盘(使用LVM动态卷管理)

三台服务器的IP地址分别为

192.168.2.171

192.168.2.172

192.168.2.173

2.1.1 服务器 192.168.2.171(dbhost, pool)¶

这台机器主要用作 数据库服务器, 运行 MySQL(Percona). 以及 后台题库,管理的 app_pool

需要安装的软件有

  • MySQL(Percona)
  • ntpdate
  • nfs-common
  • python-pip
  • virtualenv

2.1.2 服务器 192.168.2.172(nfs_server, app1)¶

这台机器主要用作 nfs_server 以及 app_server

需要安装的软件有

  • ntpdate
  • nfs-server
  • python-pip
  • virtualenv

2.1.3 服务器 192.168.2.173(ntp_server, app2)¶

这台机器主要用作 ntp_server 以及 app_server

需要安装的软件有

  • ntp
  • nfs-common
  • python-pip
  • virtualenv

2.2 创建 adaptive 用户¶

在所有的 server 上创建 adaptive 用户, 方便服务器的快速部署, 以及管理.

使用下面命令创建用户.:

$ sudo useradd -m adaptive

$ sudo passwd adaptive

-m 参数会自动创建用户目录, 记得将 adaptive 加入 sudoer 列表, 以便使用 root 权限

passwd 命令会为 adaptive 用户创建密码.

将 adaptive 用户, 加入 sudoer 组, 以便使用 sudo 权限.

修改文件 /etc/sudoers

root        ALL=(ALL:ALL) ALL
adaptive    ALL=(ALL:ALL) ALL

在 root 后加上 adaptive 相应的权限, 使用 adaptive 用户登录机器, 测试是否可以正常使用 sudo 命令

(需要分别为3台server都设置adaptive用户)

2.3 修改服务器MTU¶

由于使用默认的 MTU 值(1500), 无法使用 scp, rsync 等工具进行服务器间的文件传输.

需要修改 MTU 值到 1000 以确保这些工具能正常使用.:

adaptive@ubuntu171:~$ ifconfig

eth0      Link encap:Ethernet  HWaddr 00:1d:92:f1:e9:aa

          inet addr:192.168.2.171  Bcast:192.168.2.255  Mask:255.255.255.0

          UP BROADCAST RUNNING MULTICAST  MTU:1500  Metric:1

          RX packets:18609449 errors:0 dropped:0 overruns:0 frame:0

          TX packets:16262310 errors:0 dropped:0 overruns:0 carrier:0

          collisions:0 txqueuelen:1000

          RX bytes:2345116485 (2.3 GB)  TX bytes:3276387071 (3.2 GB)

          Interrupt:43 Base address:0xe000

手动设置:

$ sudo ifconfig eth0 mtu 1000

eth0 是网卡名称，如果你的不是这个可以做相应的更改，不过这样每次重新连接时双要设置一下，比较麻烦。

网络连接（网卡被激活时）自动设置 mtu 值

在 /etc/network/if-up.d 目录下新建一个脚本文件 setmtu

$ sudo vi /etc/network/if-up.d/setmtu

将下面的代码加入 setmtu 脚本中

#!/bin/sh

ifconfig "$IFACE" mtu 1000

保存退出，然后给这个脚本中可执行权限:

$ sudo chmod a+x /etc/network/if-up.d/setmtu

其中 $IFACE 是一个变量，返回的是当前激活的网卡名称 比如 eth0 eth1. 这样就可以在每次网卡激活时, 自动修改 MTU 值了.
